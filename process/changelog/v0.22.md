# v0.22 — Migration Hardening

**Date:** 2026-03-21

**Summary:** Refinements surfaced by running the v0.21 migration prompt with an Architect on a live downstream project (GolemUI). Adds hierarchical numbering for the action tree, a notepad branch in the knowledge tree for low-friction observation capture, and Reflecting as a third interaction mode. Removes the Developer stance (two core stances remain: Architect and Tech Lead, plus the Auditor as situational) and removes the history section from the status.md format.

**Status:** Released.

---

## Changes

### Hierarchical numbering convention

Action tree nodes use two-digit IDs starting at `05`, incrementing by `5`. This creates insertion gaps so new nodes can be added between existing ones without renumbering. Parking-lot goals (long-term, not yet scoped) count down from `95`.

Example: `05.goal.auth-redesign/`, `10.step.interaction-modes/`, `15.task.update-docs.md`.

Hierarchical addressing uses dot-separated parent-child IDs: node `10` inside goal `05` is addressed as `05.10`. This makes it possible to reference deeply nested nodes unambiguously in conversation and journal entries.

**Files changed:** `process/action-tree.md` (new Numbering Convention section, all examples updated), `process/memory.md` (naming examples), `process/conventions.md` (naming conventions).

### KT notepad branch

A `notepad/` folder at the knowledge tree root provides a low-friction scratch space for observations that surface during execution. Each notepad node is tied to a specific AT action. Any stance can write to it — no curation gate, no formality. On action completion, durable findings migrate to their domain home in the KT; the notepad node archives.

The notepad is the KT's intake buffer — the same role the journal plays for temporal observations, but scoped to an action and structured by topic rather than time. It exists because the gap between "noticed something" and "wrote a proper KT spec" was too large, and observations were being lost.

**Files changed:** `process/knowledge-tree.md` (new Notepad Branch section, updated "How the Tree Grows"), `process/action-tree.md` (action completion checklist, context file description), `process/memory.md` (context type description), `process/file-types/README.md` (context file entry), `bootstrap/README.md` (folder diagrams, mkdir commands, skeleton files), `process/journaling.md` (notepad as third recording destination, Recording by Stance tables, action completion checklist).

### Reflection mode

Reflecting is the third interaction mode alongside Planning and Executing. Use Reflecting when a significant action has just been completed and you want to evaluate what happened — what worked, what didn't, what should change for next time. Reflecting mode is retrospective: artifacts are neither provisional nor authoritative, they're analytical.

The three modes are now: Planning (shaping), Executing (building), Reflecting (evaluating).

**Files changed:** `process/principles.md` (Interaction Modes section rewritten for three modes), `process/workflow.md` (active stack Reflecting entry, task mode reference), `roles/operating-rules.md` (session open protocol), `process/action-tree.md` (status.md description, stack example, task mode reference), `process/journaling.md` (handover section), `process/conventions.md` (status vocabulary), `bootstrap/ai_readme.template.md` (mode references), `ai_readme.md` (meta-project mode references).

### Developer stance removed

The Developer stance is eliminated. Two core stances remain: Architect (designs) and Tech Lead (builds). The Auditor remains as a situational stance.

The Developer was never used in practice — across 16 GolemUI sessions and all meta-project sessions, it was never invoked independently. The Architect's specs were consistently detailed enough for direct implementation. The adaptive flow gate already made the Tech Lead situational; the Developer added a third layer that created overhead without value.

**Files changed:** `roles/developer.md` (deleted), `process/roles.md` (Developer section removed, model tiers updated), `process/workflow.md` (Developer references removed), `process/anti-patterns.md` (Developer references removed), `process/journaling.md` (Developer removed from recording matrices), `README.md` (stance descriptions updated).

### Status history removed

The `status.md` format no longer includes a History section. Historical state is tracked by the journal (temporal record) and git log (change history). The status file is purely the current state — not a changelog.

**Files changed:** `process/action-tree.md` (status.md description updated), `process/memory.md` (status.md role clarified).

## Migration Steps (v0.21 → v0.22)

### Structural changes

1. **Renumber AT nodes.** Existing AT nodes may use sequential numbering (1, 2, 3...). Renumber to two-digit IDs with gaps: `05`, `10`, `15`, etc. Update all internal references (index files, status.md, cross-references between nodes). This is the most labor-intensive step — every folder rename requires updating every file that links to it.

2. **Create KT notepad.** Create `knowledge-tree/notepad/` with a `notepad.index.md` skeleton. Create `knowledge-tree/notepad/archive/` for completed action notepads. If there's an active action, optionally create its notepad node.

3. **Remove status.md history.** If your `status.md` has a History section or table, remove it. The current state fields, active stack, and project overview are all that remain.

### Behavioral changes

4. **Developer stance.** Stop invoking the Developer. If your `ai_readme.md` or session prompts reference the Developer stance, update them. Two core stances: Architect, Tech Lead. Auditor is situational.

5. **Reflection mode.** No structural migration needed. Reflecting is now available as a third interaction mode. Use it after completing significant actions to evaluate outcomes.

### Methodology file changes

The following take effect automatically when the methodology repo is updated:

- `roles/developer.md` deleted
- `process/roles.md` updated (Developer removed)
- `process/workflow.md`, `process/anti-patterns.md`, `process/journaling.md` updated (Developer references removed)
- All three-mode references in `process/principles.md`, `process/conventions.md`, `roles/operating-rules.md`
- Bootstrap templates updated for notepad and three modes
